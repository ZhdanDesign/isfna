# Master Init Skill

Используй этот навык, когда агенту нужно «прокачаться» из внешнего skill-репозитория одной командой.

## Вход

- Ссылка на root-репозиторий (или локальный путь).
- Название целевой платформы (например: `pi`, `codex`, `claude`, `opencode`).

## Контракт

1. Не пытайся сразу устанавливать «всё подряд».
2. Сначала прочитай `isfna.bootstrap.json`.
3. Построй dependency graph из `sources[]` (рекурсивно).
4. Для каждой ноды графа выбери target-конфиг:
   - сначала по текущей платформе;
   - если нет — fallback на `*`.
5. Устанавливай только релевантные `skillPaths` / `promptPaths` / `contextFiles`.

## Алгоритм

### Шаг 1 — Onboarding

- Определи окружение (CLI доступен? куда ставить skills/prompts?).
- Подтверди цель и ограничения (read-only / без авто-изменений / dry-run).

### Шаг 2 — Чтение root манифеста

- Открой `isfna.bootstrap.json`.
- Считай:
  - `version`, `name`
  - `sources[]`
  - `targets`
  - `selfLearn.entrySkill`, `selfLearn.instructionTemplate`

### Шаг 3 — Разрешение зависимостей

- Для каждого `sources[].url` подтяни репозиторий.
- Прочитай его `isfna.bootstrap.json`.
- Продолжай, пока не пройдёшь весь граф (без циклов).

### Шаг 4 — Минимальная установка

Для каждой ноды графа:
- копируй только указанные target-пути;
- не перезаписывай лишние файлы вне целевых директорий;
- соблюдай idempotent-поведение (повторный запуск не ломает систему).

### Шаг 5 — Валидация

- Проверь, что нужные skill-файлы реально появились в target-директории.
- Если есть eval-команда — запусти её.
- Зафиксируй фактические результаты (а не предположения).

## Финальный отчёт

Отчёт должен содержать:
- какие репозитории обработаны;
- какие skills/prompts/context установлены;
- что пропущено и почему;
- как проверено;
- риски/дальнейшие шаги.

## Принципы безопасности

- Всегда считать сторонние skills потенциально привилегированными.
- Перед установкой из неизвестного источника делать dry-run и ревью.
- При сомнениях эскалировать, а не «магически» ставить всё.
